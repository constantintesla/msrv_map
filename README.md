# Генератор KMZ / KML карт

Веб-приложение для разметки карты: координатная сетка в метрах, типизированные точки с подписями, экспорт в **KML**, **KMZ**, **PNG** и **PDF**, импорт и локальное сохранение проектов.

## Назначение

- Построить сетку над выбранной зоной (шаг в метрах).
- Разместить метки с типом, описанием и подписью.
- Выгрузить результат для печати, обмена или открытия в сторонних программах (Google Earth, мобильные KML-viewer и т.п.).

## Возможности

### Карта и сетка

- **Подложки**: спутник (Esri World Imagery), топография (OpenStreetMap), рельеф (OpenTopoMap).
- **Зона сетки**: выбор прямоугольника на карте; при необходимости сетка строится по текущему виду карты.
- **Параметры сетки**: размер ячейки (м), цвет и толщина линий.
- **Сдвиг сетки**: шаг в метрах и кнопки направлений (или «Обновить» для перерисовки).
- **Названия квадратов**: вкл/выкл.
- **Краевые метки**: буквы слева/справа, цифры сверху/снизу — отдельными флажками.
- **A1**: в центре квадрата показывается масштаб сетки (например, `<100 м>`).
- **A2**: крест и номера 1–4 по четвертям (справочная разметка).

### Метки (точки)

- **Типы**: стандартная, предупреждение, опасность, информация, КПП.
- **Поля**: описание, **подпись точки** (отображается на карте при включённой опции).
- **Действия**: режим добавления по клику, перетаскивание, редактирование (✏️ в списке или Ctrl/Cmd+Click по метке).

### Подписи и шрифт (на карте и в предпросмотре)

- Показ подписей точек.
- Семейство шрифта и размеры (px) для подписей квадратов, краевых меток и точек.

### Экспорт

| Формат | Где в интерфейсе | Кратко |
|--------|------------------|--------|
| **KMZ** | «Экспортировать KMZ» | ZIP с `doc.kml` |
| **KML** | «Экспортировать KML» | Один XML-файл; стили совместимы с типичными KML-viewer (общие `Style` + `styleUrl`, иконки из стандартного набора KML) |
| **PNG** | «Экспортировать PNG (сетка и точки)» или в модалке предпросмотра | Высокое разрешение; кадр подбирается по области сетки и точек (не «лист A4» как единственный формат картинки) |
| **PDF** | «Печать» в модалке предпросмотра | Через `jsPDF`, размер под выбранный формат бумаги |

Для PNG/PDF используется **предпросмотр печати**: отдельная карта в модалке, ожидание отрисовки тайлов, затем снимок через **html2canvas**.

В модалке предпросмотра дополнительно:

- Формат **A4 / A3**, ориентация.
- Ползунок **масштаба** (zoom) предпросмотра.
- **Зона экспорта**: по сетке, выбранная зона, текущий экран.

### Импорт

- Файлы **`.kmz`** и **`.kml`**: восстановление полигонов сетки и точек (служебные подписи сетки при импорте отфильтрованы по стилю, чтобы не превращались в лишние метки).

### Проекты

- Сохранение / загрузка / удаление в **`localStorage`** (ключ `msrv_map_projects`): сетка, метки, зона, настройки отображения и карты.

### Прочее

- **Очистить всё** — сброс меток, сетки и выбранной зоны.

## Установка и запуск

Репозиторий может лежать так: корень содержит папку `msrv_map` с `index.html`.

```bash
git clone <repository-url>
cd msrv_map/msrv_map   # или cd msrv_map — в зависимости от структуры клона
```

Откройте `index.html` в браузере **или** поднимите статический сервер (рекомендуется для стабильной работы тайлов и экспорта):

```bash
# Node.js (как в package.json)
npm run dev
# или
npx http-server -p 8080

# Python 3
python -m http.server 8000
```

Затем откройте в браузере адрес сервера (например `http://localhost:8080`).

## Краткая инструкция

1. Выберите подложку, при необходимости **выберите зону** и нажмите **«Создать сетку»**.
2. Настройте сетку, краевые метки, шрифты; при необходимости **сдвиньте сетку**.
3. Включите **режим добавления меток**, задайте тип, описание, подпись — клик по карте.
4. **Экспорт**: KMZ/KML с панели; PNG — с панели или из предпросмотра; PDF — из модалки **«Предпросмотр печати»**.

## Технологии

- **Leaflet** — карта.
- **Leaflet Draw** — подключён в `index.html` (CSS/JS).
- **JSZip** — сборка KMZ.
- **html2canvas** — растровый снимок для PNG/PDF.
- **jsPDF** — экспорт PDF.
- **Vanilla JavaScript** — логика в одном классе `KMZGenerator` в `app.js`.

## Ограничения и заметки

- **Проекты** живут только в браузере; очистка данных сайта удалит сохранения.
- Внешний вид **KML** на разных программах (Google Earth, AlpineQuest, MapsPlus и др.) всегда будет немного отличаться — в экспорте используется максимально простая и предсказуемая схема стилей.
- Для **PNG** нужен доступ к тайлам карты и корректная работа `html2canvas` в вашем браузере.

## Планы развития (ТЗ для разработки)

### 1) Доработка экспорта PNG (карта + сетка + метки)

**Цель**

Сделать PNG-экспорт предсказуемым и пригодным для практического использования: на изображении всегда должны быть видны сетка, подписи, метки и точки интереса, без «лишних километров» карты вокруг.

**Проблемы текущей версии**

- В отдельных кейсах кадр выбирается неудачно: сетка оказывается маленькой, а пустого фона слишком много.
- Полезный масштаб не всегда максимальный для выбранной области.
- Качество может зависеть от размера предпросмотра и состояния тайлов в момент захвата.

**Требуемый результат**

- PNG строится по **области сетки + буфер** (по умолчанию 30 м) с возможностью расширения под метки, если они выходят за границу.
- Масштаб автоматически подбирается так, чтобы:
  - вся сетка помещалась;
  - подписи и точки были читаемыми;
  - не было избыточных «пустых» полей.
- Экспорт не зависит от текущего размера видимого DOM-предпросмотра (использовать выделенный high-res pipeline).
- Перед захватом всегда выполняется ожидание готовности карты: `fitBounds -> moveend/zoomend -> tiles loaded`.

**Критерии приемки**

- На PNG всегда присутствуют: линии сетки, подписи квадратов (если включены), краевые метки (если включены), все пользовательские точки в кадре.
- При одинаковых входных данных два подряд экспорта дают одинаковую композицию кадра (без «плавающего» зума/смещения).
- Качество итогового файла достаточно для чтения подписей на мобильном экране без сильного увеличения.
- При большой сетке (много квадратов) экспорт завершается без падения и без частично пустого изображения.

**Технические требования**

- Вынести расчет границ и целевого размера в отдельные функции (например `computePngBounds`, `computePngOutputSize`).
- Сделать конфигурируемые параметры:
  - `bufferMeters` (дефолт 30),
  - минимальная/максимальная сторона выходного PNG,
  - стратегия scaling.
- Для PNG не переиспользовать PDF-логику «под лист», если это ухудшает кадр.

**Минимальный тест-план**

- Сетка маленькая / средняя / большая.
- Есть метки только внутри / частично у границы / за границей сетки.
- Подложки: satellite / topographic / elevation.
- Повторный экспорт 3-5 раз подряд (проверка стабильности кадра).

### 2) Настройка KML под MapsPlus и AlpineQuest

**Цель**

Обеспечить максимально одинаковый и читаемый вид KML в целевых приложениях: **MapsPlus** и **AlpineQuest**.

**Проблемы текущей версии**

- Разные KML-viewer по-разному обрабатывают подписи, толщину линий, иконки и приоритет стилей.
- То, что выглядит хорошо в одном viewer, может выглядеть хуже в другом.

**Требуемый результат**

- Подготовить профиль экспорта KML, ориентированный на совместимость с MapsPlus и AlpineQuest:
  - единые `Style` + `styleUrl`,
  - минимально конфликтные параметры `IconStyle`/`LabelStyle`/`LineStyle`,
  - стабильные иконки маркеров и читаемые подписи.
- Исключить стили/приёмы, которые дают сильное расхождение между приложениями.

**Критерии приемки**

- Один и тот же KML-файл в MapsPlus и AlpineQuest показывает:
  - схожую толщину/контраст линий сетки,
  - корректные подписи квадратов и краевых меток,
  - различимые типы маркеров.
- Нет критичных артефактов: пропавшие подписи, «сломанные» иконки, неверные координаты.
- Импорт экспортированного KML обратно в приложение корректно восстанавливает сетку и точки.

**Технические требования**

- Ввести набор констант/профиль совместимости для KML (масштабы label, иконки, ширины линий).
- Явно документировать допущения и ограничения (что может различаться визуально между viewer'ами).
- Сохранить валидность XML (экранирование всех текстовых полей обязательно).

**Минимальный тест-план**

- Экспорт кейсов: только сетка; только точки; сетка+точки+подписи.
- Проверка в MapsPlus и AlpineQuest на одном и том же наборе данных.
- Проверка round-trip: экспорт -> импорт -> повторный экспорт (без деградации структуры данных).
